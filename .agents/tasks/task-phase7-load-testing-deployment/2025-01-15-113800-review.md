## CI cache vs npm ci interaction

The CI workflow in both `lint-typecheck` and `test` jobs does:

```yaml
- name: Cache node_modules
  uses: actions/cache@v4
  with:
    path: node_modules
    key: ${{ runner.os }}-node-${{ hashFiles('package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-node-
- run: npm ci
```

`npm ci` removes the existing `node_modules` directory before installing. On cache hit, the restored directory is immediately destroyed. On cache miss, `npm ci` installs fresh and the cache action saves the result — but the next run hits the cache-hit scenario and gains nothing. The fix:

```yaml
path: ~/.npm
key: ${{ runner.os }}-npm-${{ hashFiles('package-lock.json') }}
```

Caching `~/.npm` means `npm ci` can skip downloading tarballs from the registry on cache hit, cutting install time by 30-60% depending on dependency count. This is a minor CI performance issue, not a blocker.
